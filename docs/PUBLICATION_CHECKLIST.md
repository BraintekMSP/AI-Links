# Publication Checklist

## Purpose

Use this checklist before publishing an adapted version of this framework publicly.

## Review items

- confirm no client names remain
- confirm no internal credentials or secrets remain
- confirm no private file paths remain
- confirm example repos and prompts are generic
- confirm destructive guidance is framed safely
- confirm temporary internal notes have been removed
- choose and add a license
- review `SECURITY.md`
- review `CONTRIBUTING.md`

## Optional extras

- issue templates
- code of conduct
- example adaptation repos
- release tags

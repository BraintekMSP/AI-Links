# AI-Links

`AI-Links` is a public-safe framework for AI-assisted software delivery.

It is meant for teams that want:

- safer AI project intake
- clearer repo-local guardrails
- measurable repo maturity
- explicit cross-repo contracts
- subagent-safe collaboration patterns

This is not an application runtime.
It is a control-plane and documentation scaffold that clients can adapt to their own repos.

## What is included

- a generic AI startup prompt
- a reusable `AGENTS.md` model
- the portable `AGENTS-schema-*` family
- a repo `2.5` readiness model
- a cross-repo contract model
- a subagent safety model
- a documentation cleanup method
- ready-to-copy templates

## Who this is for

- internal software teams
- MSPs and IT teams
- consultants building client-specific delivery workflows
- organizations using AI agents or assisted contributors without wanting chaotic repo behavior

## Startup path

1. Read `AGENTS.md`

## Core idea

Good AI collaboration depends on four things:

1. startup truth is obvious
2. behavior truth stays in code and explicit contracts
3. docs own policy, runbooks, and boundaries
4. destructive or drifting agent behavior is prevented, not merely discouraged
